# Smart Import Feature Guide

## Overview

The **Smart Import** feature allows photographers to paste any questionnaire data from couples—exactly as provided—without worrying about formatting. The system intelligently parses and organizes the information automatically.

## How to Use Smart Import

### Step 1: Copy the Data
1. Open the couple's questionnaire, email, or Google Form response
2. Select all the text they provided
3. Copy it (Ctrl+C / Cmd+C)

### Step 2: Open Smart Import
1. Click the "Smart Import" button in your photographer dashboard
2. You'll see a large text area

### Step 3: Paste the Data
1. Click in the text area
2. Paste the raw data (Ctrl+V / Cmd+V)
3. Don't worry about formatting!

### Step 4: Click "Smart Import" Button
1. The system processes the data
2. You'll see a preview of what was extracted
3. Review the results

### Step 5: Confirm or Adjust
1. If everything looks good, click "Import Data"
2. The wedding data is populated in your shotlist
3. You can edit any details before generating the shot list

## Technical Details

### What the Parser Does

1. **Normalizes Text**
   - Fixes line breaks (CRLF → LF)
   - Removes extra spaces
   - Standardizes tabs

2. **Identifies Sections**
   - Recognizes headers (Bridesmaids, Family, etc.)
   - Groups related content
   - Categorizes by type

3. **Extracts People**
   - Parses names and relationships
   - Handles multiple formats
   - Cleans up special characters

4. **Organizes Data**
   - Side 1 vs Side 2 attendants
   - Family members by partner
   - Portrait requests
   - Special groupings

5. **Validates Results**
   - Checks for minimum required data
   - Reports any parsing issues
   - Provides helpful warnings

### Input Parsing Rules

- **"Name - Relationship"** → Name: "Name", Relationship: "Relationship"
- **"Name (Relationship)"** → Name: "Name", Relationship: "Relationship"  
- **"Name"** → Name: "Name", Relationship: "Guest"
- **Bullet points removed** → Content parsed normally
- **Dashes and asterisks stripped** → Content extracted

### Section Recognition

Headers like these are recognized:
- "Bridesmaids", "Bride's attendants", "Side 1 attendants"
- "Groomsmen", "Groom's attendants", "Side 2 attendants"
- "Bride's family", "Groom's family", "Parents", "Family"
- "Requested portraits", "Portrait requests", "Special photos"
- "Special groupings", "Custom groupings"

## Troubleshooting

### Problem: Names not parsing correctly

**Example input:**
```
SOOST, Katie - Maid of Honor
```

**Solution:**
The system expects "First Last - Role". If your input is different, you can still import and edit afterwards.

### Problem: Relationship not recognized

**Example:**
```
Katie - Step-Sister of Bride
```

**Solution:**
The system will extract it as is. During import, you can edit to a standard relationship type like "Step-Sister" if you prefer consistency.

### Problem: Multiple lines for one person

**Example:**
```
Katie Soost
Maid of Honor
```

**Solution:**
This might not parse as a single entry. For best results, use: "Katie Soost - Maid of Honor"

### Problem: Table format not parsing well

**Example:**
```
| Katie Soost | Bridesmaid |
```

**Solution:**
Copy the table data and paste it. The parser will extract the names and roles.

## Best Practices

1. **Use Clear Formatting**
   - "Name - Relationship" format works best
   - Separate sections clearly
   - Use consistent capitalization

2. **Include Relationships**
   - Always include the role (Maid of Honor, Father, etc.)
   - Helps the system categorize correctly

3. **Review Before Import**
   - Check the preview carefully
   - Make sure all names are present
   - Verify relationships look correct

4. **Edit After If Needed**
   - Don't worry about perfection
   - You can adjust anything in the editor
   - The system is flexible

5. **Use Descriptive Roles**
   - "Maid of Honor" instead of "MOH"
   - "Father" instead of "Dad"
   - Standard terms help with organization
